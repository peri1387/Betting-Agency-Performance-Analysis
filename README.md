# 🎰 Betting Agency Performance Analysis

## 📌 Project Overview
This project is a comprehensive data analysis study for a gambling agency. Combining my 12 years of experience as an OPAP agency owner with modern Data Analytics tools, I created an end-to-end pipeline that answers critical business questions.

### 🛠 Tech Stack
- **Python (Faker & Pandas):** Creating synthetic data based on real market business rules.

- **MySQL:** Storing and processing 10,000+ transactions. Using CTEs and Aggregations to calculate profitability and commissions.

- **Power BI:** Designing Dashboard for decision-making.

### 📁 Repository Structure
- `/data`: CSV files generated by the Python engine and results from queries.
- `/scripts`: Python automation for data generation.
- `/sql`: SQL scripts for creating reporting views and schema management.
- `/reports`: The final .pbix file with a pdf file and project screenshots.

### 🔍 Business Questions Answered
- Which game is the "pillar" of turnover and which of net commission?

- How does the type of customer (VIP vs Regular) affect profitability?

- What are the monthly turnover trends and how can we predict staffing needs?

### 📊 Key Insights
- Revenue vs Commission: The game "Joker" may not have the highest trading volume, but due to the higher commission it constitutes 21% of the net profit.

- Customer Behavior: "Occasional" players bring in the highest total volume (33K), indicating a need for marketing actions to convert them into "Regulars".

- Seasonality: A drop in turnover is observed in February and November, which allows for scheduling staff leave.
